Motion Capture System

AMD-Sponsored Project 2025-2026 TUSAŞ Lab @ YTU

Low-cost motion capture alternative using AMD Kria KV260 SoC with FPGA-accelerated blob detection and confidence-weighted Kalman filtering. Achieved sub-centimeter accuracy at a fraction of commercial system cost.

AMD Kria KV260 VHDL C++ MIPI CSI Computer Vision Kalman Filter DLT/SVD Vivado

Overview

Commercial motion capture systems like Vicon cost $50,000+ and require specialized setups. This project demonstrates that comparable accuracy can be achieved using FPGA-based processing and computer vision algorithms at a dramatically lower cost.

The system uses the AMD Kria KV260 SoC, which combines programmable logic (FPGA fabric) with ARM processing cores, enabling real-time hardware acceleration of vision algorithms alongside algebraic reconstruction on the CPU.

System Architecture

Hardware Pipeline

  • Camera Interface: OV5640 MIPI CSI cameras capture synchronized frames
  • PL (FPGA) Processing: Hardware-accelerated blob detection identifies marker centroids in real-time
  • PS (ARM) Processing: CPU runs DLT/SVD triangulation to reconstruct 3D positions
  • Post-Processing: Confidence-weighted Kalman filter smooths trajectories and reduces noise

Key Technical Decisions

The design splits compute-intensive tasks between PL and PS based on their characteristics:

  • Blob detection runs on FPGA fabric (PL) — highly parallelizable, pixel-level operations benefit from custom hardware
  • Triangulation runs on ARM cores (PS) — matrix operations leverage optimized BLAS libraries
  • AXI stream interface transfers centroid coordinates from PL to PS with minimal latency

Implementation Details

FPGA Blob Detection

Custom VHDL pipeline processes 640×480 frames at 60 FPS. Connected component labeling identifies marker regions, centroid calculation uses streaming architecture to minimize BRAM usage.

DLT Triangulation

Direct Linear Transform reconstructs 3D points from multiple 2D camera views. SVD decomposition solves overconstrained system. Optimized using Eigen library for ARM NEON instructions.

Kalman Filtering

Confidence-weighted filter adapts measurement noise based on detection quality. Reduces jitter while maintaining responsiveness to genuine motion. ~18% RMSE improvement over raw measurements.

Camera Calibration

Zhang's method for intrinsic parameters, bundle adjustment for extrinsics. Calibration accuracy directly impacts reconstruction precision — achieved <0.5px reprojection error.

Results

  • Accuracy: Sub-centimeter precision in 1m³ capture volume
  • Latency: <20ms end-to-end (camera → 3D output)
  • Throughput: 60 FPS tracking of 4+ markers simultaneously
  • Cost: ~$500 hardware vs. $50k+ commercial systems

Conference paper submitted to SIU 2026 detailing the confidence-weighted Kalman filter approach and system architecture.

Challenges & Solutions

Clock Domain Crossing

MIPI CSI pixel clock doesn't align with AXI stream clock. Implemented dual-clock FIFO with handshaking to safely transfer data between domains. Gray code counters prevent metastability.

Memory Bandwidth

Streaming architecture processes pixels in raster order without frame buffering. Reduces DRAM bandwidth from 640×480×60 = 18.4 MB/s to just centroid coordinates (~100 bytes/frame = 6 KB/s).

Calibration Stability

Initial calibration degraded over time due to thermal drift and vibration. Added periodic recalibration routine using known reference points in the capture volume.

Future Work
